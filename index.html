Introduction

This document specifies how to interact with the Helden der Volksmusik platform, including how to request resources, authenticate and how to post data to the backend. All operations against the platform are done against a RESTful API and all resources are accessible via HTTPS only.

Conventions

  • Unless expressed otherwise, all dates are expressed in number of milliseconds since January 1st, 1970 in GMT.
  • Authentication and session preservation is handled by session cookies. Every HTTP request to the site will deliver you a session cookie. Make sure to attach this cookies to all your HTTP requests.
  • All POST, PUT and DELETE HTTP operations (verbs) will require you to submit a csrf token. Use the endpoint /token to get one programatically. If you fail to submit a csrf token with these HTTP verbs, or submit an invalid one, you will receive an HTTP 500 error with data payload EBADCSRFTOKEN. If you get this error, we recommend you to get a csrf token by calling the /token endpoint and retry the operation.
  • The preferred way to submit a csrf token with your request is to attach it to the url via que URL query _csrf=. You can also add it as a cookies with the name x-csrf-token.
  • Some endpoints share the same url as some webpages. Therefore, in order to guarantee that you will received data (instead of an HTML response), we advise you to add the HTTP header X-Requested-With: XMLHTTPRequestto every request.
  • The media endpoints that store video and audio files support streaming. When calling these files, expect to receive partial content responses (HTTP 206). That is, it is safe to start playback of such files immediately after your media player has buffered enough data.
  • This documentation describes the data models (resources) and the properties that pertain to them. You may receive response with additional properties to the ones listed here. We discourage you to create a dependency on properties received within the object but not documented here, as we might rename or remove them from the object model in future releases.

Resources

Post

The post is the core of the platform and represents content uploaded by artists themselves.

Property Description
content String. Main content text of the post
date_published Int64. Date, expressed in milliseconds, when the post was created
liked_by Array. List of userIds that have liked this post
comments Array. List of comment objects that have been written for this post. See Comment resource for details
title String. Title of the post, usually the title of the song
likesCount Int. Number of times the post has been liked.
commentsCount Int. Number of times the post has been commented.
viewsCount total number of times the post has been viewed.
media JSON. Information about the media assets associated with this post. See Media object for information
styles Array. Music genre category of this post, as provided by the artist. Can be null or empty
category String. Category assigned to this post, as provided by the artist. Can be null or empty
cuid String. This is the main, unique identifier of the post
authorId String. Unique userId of the author of the post
isHitParadeEligible Boolean. Specifies whether this post is participating in the hit parade or not
isEncoded Boolean. If true, it means the main media asset of this post has already been processed by our encoding service and it is safe to stream it. If false, it means our encoding service has not yet optimised this video for web streaming, and thus this post should not be displayed to the user
id in disuse
author JSON. User object with information about the author of the post. See User resource description for details.

User

Provides information about a user

Property Description
userId String. Unique ID of the user object in the platform
displayName String. Name to be used as display string. We encourage you to use this value, instead of concatenating properties to build the name yourself
city String. Name of the user's city
country String. Name of the user's country
role String. Specifies where the user is an artist or a fan
dateCreated Int64. Date, in milliseconds, when the user registered in the platform
canopyPicture String. Absolute uri of the user's canopy picture
selfie String. Absolute uri of the user's profile picture
identityProvider String. If the value is local, it means the user has registered with own email address and password. Otherwise, specifies the social network with which the user registered (for example, Facebook)
identityProviderId String. If the user registered via a social network, this property specifies the unique userId provided by the social network for the Helden app
points Int. Specifies the number of points the user has accumulated in the platform
givenName String. Given name of the user
lastName String. Last name (family name) of the user

Media

The media object contains information about the media assets associated with a post.

Property Description
uri String. Absolute uri to the file that contains the main media of the post
contentType String. Specifies the MIME type of the uri asset
poster String. Absolute uri of the poster (image) file. A poster is a screenshot from a frame of a video. Can be null.
webm String. Absolute uri of the file that contains the webm version of this video. Can be null or empty string.
mp3 String. Absolute uri of the audio file for this post. Can be null or empty string. This applies to video and audio posts.

Endpoints

GET /post?continuation_token=[token]

Requires authentication: No Return Array of Post objects This endpoint will return up to 5 Posts, order from newest to oldest (as specified in the property date_published. In order to assemble a series of posts chronologically, you should store the value of date_published for the last post received in a response (the oldest post) and use this value in the query parameter continuation_token to retrieve the next 5 older posts.

Parameter Description
continuation_token Date in milliseconds. If submitted, the API will return the next 5 posts older than the value of continuation_token

GET /post/[postId]

Returns a Post JSON object for the requested :postId.

Parameter Description
postId Unique ID of the post to be requested. This corresponds to the value of property cuid of the post object.

GET /post/random

Returns a random Post object This endpoint is particularly useful for the radio feature on the iPhone app, as the radio offers the user playback of random songs. Use this endpoint to get a random Post from our entire catalog

GET /post/[postId]/comments/

Returns an array with all comments that belong to a specific post. See Comment object model for further details

Parameter Description
postId Unique ID of the post to be requested. This corresponds to the value of property cuid of the post object.

GET /sas

Requires authentication This endpoint will return a Shared Access Signature token. This token is required when posting media assets to our blob storage endpoint. These token has a validity of up to 15 minutes, so your application must control for expired tokens and request a new one if needed.

Example token "se=2015-09-20T23%3A07%3A53Z&sp=w&sv=2014-02-14&sr=c&sig=8SF5CMT5%2F5xRmIiDIF3J7q5LUx90gL9ftFPIMHQcXvw%3D"

GET /token

Retrieves a CSRF token. This endpoint will respond with a cookie named x-csrf-token. Store its value, as this is the corresponding csrf token for the sessionID with which you are connecting

GET /search

This endpoint will deliver a read-only API Key for our search service. Use this API to perform a search on our search provider. If you do not provide this API key to the search provider, you will receive an HTTP 403 error. The data payload will contain the API key.

Sample key: "C01512F72E1353042C8FEB33DB9E7929"
