Introduction

This document specifies how to interact with the Helden der Volksmusik platform, including how to request resources, authenticate and how to post data to the backend. All operations against the platform are done against a RESTful API and all resources are accessible via HTTPS only.

Conventions

  • Unless expressed otherwise, all dates are expressed in number of milliseconds since January 1st, 1970 in GMT.
  • Authentication and session preservation is handled by session cookies. Every HTTP request to the site will deliver you a session cookie. Make sure to attach this cookies to all your HTTP requests.
  • All POST, PUT and DELETE HTTP operations (verbs) will require you to submit a csrf token. Use the endpoint /token to get one programatically. If you fail to submit a csrf token with these HTTP verbs, or submit an invalid one, you will receive an HTTP 500 error with data payload EBADCSRFTOKEN. If you get this error, we recommend you to get a csrf token by calling the /token endpoint and retry the operation.
  • The preferred way to submit a csrf token with your request is to attach it to the url via que URL query _csrf=. You can also add it as a cookies with the name x-csrf-token.
  • Some endpoints share the same url as some webpages. Therefore, in order to guarantee that you will received data (instead of an HTML response), we advise you to add the HTTP header X-Requested-With: XMLHTTPRequestto every request.
  • The media endpoints that store video and audio files support streaming. When calling these files, expect to receive partial content responses (HTTP 206). That is, it is safe to start playback of such files immediately after your media player has buffered enough data.

Resources

Post

The post is the core of the platform and represents content uploaded by artists themselves.

Property Description
content String. Main content text of the post
date_published Int64. Date, expressed in milliseconds, when the post was created
liked_by Array. List of userIds that have liked this post
comments Array. List of comment objects that have been written for this post. See Comment resource for details
title String. Title of the post, usually the title of the song
likesCount in disuse
commentsCount in disuse
viewsCount total number of times the post has been viewed.
media JSON. Information about the media assets associated with this post. See Media object for information
